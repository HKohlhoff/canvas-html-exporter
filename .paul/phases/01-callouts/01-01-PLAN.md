<objective>
## Goal
Add Obsidian callout rendering to `markdownToHtml()` so that `> [!TYPE] Title` blocks are exported as styled callout boxes instead of plain blockquotes.

## Purpose
Obsidian callouts are a core content type used widely in canvas notes. Without callout support, exported HTML silently loses the visual structure and semantic meaning of callout blocks — breaking the fidelity goal.

## Output
Modified `canvas-exporter/src/converter.ts` with:
- Callout detection + HTML generation inside `markdownToHtml()`
- Callout CSS in the canvas node inline styles (inside `convertCanvasToHtml()`)
- Callout CSS in the standalone markdown page styles (inside `buildMarkdownDocumentHtml()`)
</objective>

## Project Context
@.paul/PROJECT.md

## Source Files
@canvas-exporter/src/converter.ts

<acceptance_criteria>

## AC-1: Callout blocks render as styled divs
```gherkin
Given a markdown text node or .md file containing `> [!NOTE] My Title\n> Content here`
When the canvas is exported to HTML
Then the output HTML contains a `<div class="callout callout-note">` with a title bar showing "My Title" and a content area showing "Content here"
```

## AC-2: Callout title defaults to type name when omitted
```gherkin
Given a callout block with no custom title: `> [!WARNING]\n> Watch out`
When exported
Then the callout title bar shows "Warning" (capitalized type name)
```

## AC-3: Plain blockquotes are unaffected
```gherkin
Given a standard blockquote: `> This is a quote`
When exported
Then the output is a plain `<blockquote>` element, unchanged from current behavior
```

## AC-4: Callout types map to distinct colors
```gherkin
Given callouts of types note/info, tip/success, warning, danger/error, example, quote
When rendered
Then each type group has a distinct border color and title background color
```

  <name>Task 1: Add callout detection and HTML generation to markdownToHtml()</name>
  <files>canvas-exporter/src/converter.ts</files>
  <action>
    In the blockquote handler inside `markdownToHtml()` (around line 718–735), BEFORE rendering the `<blockquote>`, check if the first collected quote line matches the callout pattern:

    ```
    /^\[!([\w-]+)\](?:\s+(.*))?$/i
    ```

    If it matches:
    - Extract `type` = match[1].toLowerCase()
    - Extract `title` = match[2]?.trim() || (type[0].toUpperCase() + type.slice(1))
    - Remove the first line from `quoteLines` (it's the `[!TYPE]` header)
    - Render remaining lines through `markdownToHtml(quoteLines.join("\n"))`
    - Output:
      ```html
      <div class="callout callout-{type}">
        <div class="callout-title">{escapeHtml(title)}</div>
        <div class="callout-content">{inner}</div>
      </div>
      ```

    If it does NOT match: existing `<blockquote>{inner}</blockquote>` behavior unchanged.

    The detection must happen after collecting all quote lines but before rendering.
    Use `escapeHtml()` for the title. Use the existing `markdownToHtml()` call for inner content.
    Avoid: adding new helper functions — inline the detection logic.
  </action>
  <verify>

    Inspect the compiled output in `release/main.js` for the callout class names.
  </verify>
