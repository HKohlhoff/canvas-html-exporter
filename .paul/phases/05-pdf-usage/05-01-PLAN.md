<objective>
## Goal
PDF canvas nodes in the exported HTML show a readable, scrollable iframe with a clickable title header that opens a dedicated viewer page.

## Purpose
Currently PDF nodes embed an iframe with `sandbox="allow-same-origin"` which blocks browser PDF rendering, show no title, and have a small fallback link instead of a proper header. This plan fixes all three gaps.

## Output
- `exporter.ts`: generates `{pdfname}-viewer.html` for each PDF node; sets `canvasHref` on the node
- `converter.ts`: replaces fallback link with title link, removes broken sandbox, adds CSS for `.pdf-title-link`
</objective>

<task type="auto">
  <name>Task 2: Fix PDF rendering in converter.ts — sandbox, title link, CSS</name>
  <files>canvas-exporter/src/converter.ts</files>
  <action>
    **A. Fix renderNodeContent() for PDF nodes** (around line 733):

    Replace current PDF branch with:
    ```typescript
    if (node.fileKind === "pdf") {
      const pdfHref = escapeAttribute(node.exportPath || node.file || "");
      if (!pdfHref) return "<p>Leerer PDF-Knoten</p>";
      const viewerHref = escapeAttribute(node.canvasHref || pdfHref);
      const pdfTitle = escapeHtml(node.displayName || node.file || "PDF");
      const pdfTitleAttr = escapeAttribute(node.displayName || node.file || "PDF");
      return `<div class="pdf-embed"><a class="pdf-title-link" href="${viewerHref}" target="_blank" rel="noopener noreferrer"><div class="pdf-title">${pdfTitle}</div></a><iframe src="${pdfHref}" title="${pdfTitleAttr}" loading="lazy"></iframe></div>`;
    }
    ```

    Key changes:
    - iframe `src` uses `node.exportPath` (raw PDF), NOT `node.canvasHref`
    - Title link `href` uses `node.canvasHref` (viewer page), falls back to raw PDF if not set
    - `sandbox` attribute removed entirely (was blocking browser PDF viewer which requires JS)
    - `.pdf-fallback-link` removed (replaced by title header)

    **B. Update CSS in both style blocks** (canvas CSS ~line 154 and markdown page CSS):


    Avoid: adding `sandbox` back in any form — its absence is intentional.
    Avoid: changing `overflow: hidden` on `.node.pdf .node-content` — the outer clip is correct; the iframe scrolls internally.
